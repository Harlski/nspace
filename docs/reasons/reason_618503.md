# Reason 618503 — companion rationale for `THE-LARGER-SYSTEM` edits

**Linked change:** [THE-LARGER-SYSTEM.md](../THE-LARGER-SYSTEM.md) — mandatory workflow to add `docs/reasons/reason_{unique_6digit_id}.md` whenever that document is updated on purpose.

## Why the system was updated

`THE-LARGER-SYSTEM` is meant to stay small and durable, but **principles without provenance** rot: readers cannot tell whether a line was careful consensus, a one-off mood, or obsolete context. Requiring a separate rationale file for each intentional edit creates a **cheap, grep-friendly audit trail** that does not bloat the principles doc itself.

## How this better supports THE-LARGER-SYSTEM

- **Clarity of intent:** the main doc stays focused on *what we believe*; each `docs/reasons/reason_XXXXXX.md` file records *why we changed belief* and *which problem the change optimizes for*.
- **Evolvable coherence:** future work can read rationales when a principle seems wrong, instead of silently rewriting history.
- **Scales with incompleteness:** the larger system is explicitly incomplete; rationales make partial evolution safer by documenting trade-offs at decision time.

This file exists to satisfy the same rule it introduces (dogfooding the workflow).
