# reason_407318 — Daily (UTC) reset for the World Cup tally, with retained history

## Why THE-LARGER-SYSTEM was updated

The World Cup country tally was the first **competitive, player-facing leaderboard** in
Nimiq Space, and it was originally a single cumulative store. Making it reset **daily at
00:00 UTC** is a meaningfully different shape of state — recurring, time-bucketed, and
visible to players — that touches "long-lived player expectations." A naive daily reset
that simply clears the tally would silently destroy results and could mis-fire under clock
skew or restart timing. Because this is the first instance of a recurring competitive
reset (and a template future leaderboards are likely to copy), it warrants a recorded
decision rather than living only in `scoreStore.ts`.

## How it serves the larger system

- **Non-destructive by construction:** Each completed day is archived to `history` before
  the live bucket resets, and the previous winning day is retained as `prevWinner`. The
  request was explicitly to "keep historic data for later," so resets only ever *move*
  data, never delete it. Legacy cumulative stores migrate into a `0000-legacy` history
  entry instead of being dropped.
- **Identity outlives scores:** A player's chosen country lives in persistent `profiles`;
  only goal *counts* reset. Players never have to re-pick their flag each day, which keeps
  the daily cadence from feeling punitive.
- **Safety / determinism:** Rollover is **forward-only** (`rolloverIfNeeded`, keyed by
  `utcDayKey`), so clock skew, replays, or out-of-order timestamps can never archive a live
  day or blank the champion. A day with no goals leaves the previous champion intact rather
  than clearing the crowd's flag.
- **Server authority, cosmetic client flourish:** The reset is driven in-process on the
  existing room tick (no cron, no new service) and broadcast to field clients. The previous
  day's champion only feeds a **purely cosmetic** crowd flag (banners + flag-biased cheer
  emoji), keeping seasonal spectacle on the client and authoritative scoring on the server.
- **Reversibility:** Still flag-isolated under `worldcup/`; turning the flag off or deleting
  the feature removes the reset, the broadcast, and the crowd flag together. The pattern
  (archive + persistent identity + forward-only rollover) is the recommended default for any
  future recurring leaderboard.
