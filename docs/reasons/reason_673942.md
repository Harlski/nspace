# Reason 673942 — Patch notes norm + release guide

## Why `THE-LARGER-SYSTEM.md` was updated

We added an explicit **public patch notes** norm (what good copy optimizes for, recommended order, tier roles) and linked **`docs/patchnotes-release.md`** as the standing **editorial guide** for `patchnote/versions/*/public/*.md`. The in-app `/patchnotes` surface should stay aligned with how we ask humans and agents to write release copy.

## How this serves the larger system

- **Coherence** — One outline for all tiers (brief → developers) avoids drift between Discord, in-app copy, and repo markdown.
- **Evolvability** — Optional **`[NEW]` / `[FIX]` / …** tags are documented in one place; the client renders them as badges so categories stay consistent without a second data source.
- **Clear direction** — Public notes stay **short and outcome-first**; internal discussion and deep inventories stay in **`reasons.md`** and other technical docs.
