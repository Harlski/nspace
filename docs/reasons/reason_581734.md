# reason_581734 — Floor SV color picker (full spectrum)

## Why update THE-LARGER-SYSTEM

Floor painting previously shared the **hue-only ring** with blocks. That locked authors to one mid-tone per hue unless they typed hex, which fights the product goal of expressive per-tile floors. We shipped a **rectangular SV + hue-strip** control for **Room → Floor** first (Option A), while leaving object/sky on the ring.

## How it serves the larger system

- **Tiles:** Clarifies that floor `colorRgb` authoring is full-spectrum (including light/dark), not “same ring as blocks.”
- **Authoring HUD forward constraint:** Documents when to prefer a rectangular spectrum picker vs the compact hue ring, so later surfaces (blocks, sky) can migrate deliberately without inventing a third color UX language.
- **Wire format unchanged:** Still `colorRgb`; only the client control changes.
