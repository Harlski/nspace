# Reason 770142 — Header marquee in THE-LARGER-SYSTEM

**Linked change:** [THE-LARGER-SYSTEM.md](../THE-LARGER-SYSTEM.md)

## Why `THE-LARGER-SYSTEM.md` was updated

The header marquee shipped as a **cross-cutting** feature: persisted admin settings, public HTTP API, streak store, and client HUD animation. Without a short normative note, future work could split **authority** the wrong way (e.g. driving scroll cadence from the server) or regress **rotation semantics** tied to a real CSS loop.

## What changed

A **“Public HUD messaging (header marquee)”** subsection under *Recorded decisions & forward constraints* states:

- **Server:** visibility, copy, leaderboard rows (dedupe / disambiguation), dwell seconds, clamped streak fallback seconds.
- **Client:** duplicated ticker, `animationiteration` for streak ↔ message handoff, resize/image remeasure, and invisible per-chunk padding when content is narrower than the viewport so one loop still covers the full visible band.

A **Direction** line caps payload growth expectations.

## How this serves the larger system

- **Coherence:** One place describes who owns **what** for this HUD surface.
- **Evolvability:** Clear split lets the server evolve stores/APIs while the client improves motion and accessibility without renegotiating gameplay authority.
- **Direction:** Keeps hot paths small and avoids accidental “server tick for pixels” designs.
