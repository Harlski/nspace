# reason_920418 — THE-LARGER-SYSTEM: teleporter Landing Hint + preview

## Why this update

Teleporters had a structural mismatch: cross-room destinations were stored as absolute tiles but authored only from the source room's map and build radius. The overhaul makes **Landing Hint + Join Spawn fallback** the product model and adds an **editor-authorized room preview** for picking destinations in other rooms.

## What changed in `THE-LARGER-SYSTEM.md`

New subsection **Teleporters: Landing Hint + cross-room preview** — configure permissively, resolve at warp, preview overlay for cross-room pick, atomic linked-pair moves, Hub fixed spawn.

## How this serves the larger system

- **Coherence** — Aligns wire format, UI, and glossary ([CONTEXT.md](../../CONTEXT.md)) on hints vs guarantees.
- **Evolvability** — Layout snapshot API is reusable for other editor previews; teleporter rules are explicit for future room types.
- **Player expectations** — Room owners can change layouts without silently breaking teleporters; warpers land at Join Spawn instead of inside blocks.
