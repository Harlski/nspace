# Reason 291847 — Release line in THE-LARGER-SYSTEM

**Linked change:** [THE-LARGER-SYSTEM.md](../THE-LARGER-SYSTEM.md)

## Why `THE-LARGER-SYSTEM.md` was updated

Contributors needed a **single normative place** for how Nimiq Space ties **semver** and **patch-note folders** to the act of merging to `main`, without relying only on `patchnote/README.md` procedural detail.

## What changed

A short **“Release line: patch notes + semver on merge”** subsection was added: the cue **“Prepare for merge”** maps to **`npm run prepare-merge`**, which renames `UNRELEASED`, bumps root `package.json`, and seeds the next `UNRELEASED` tree so the following commit/push carries the version **in-repo** by design.

## How this serves the larger system

- **Coherence:** One command keeps `package.json` and `patchnote/versions/<version>/` aligned.
- **Evolvability:** Clear hook for later CI (e.g. git tags) without changing the author workflow today.
- **Direction:** Agents and humans share the same merge gate language (“Prepare for merge”).
