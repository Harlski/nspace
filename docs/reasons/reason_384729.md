# Reason 384729 — Hotfix tier + skill in the larger system

## Why `THE-LARGER-SYSTEM` was updated

We introduced an optional **`Hotfix`** audience in the in-app **`/patchnotes`** feed (`public/04-hotfix.md`) and documented how it differs from the normal Brief → Developers tiers. We also added a **Cursor skill** ([`.cursor/skills/hotfix-release-notes/SKILL.md`](../../.cursor/skills/hotfix-release-notes/SKILL.md)) so agents and humans use one workflow for urgent corrective release copy.

## How this serves the larger system

- **Clear product communication:** Routine patch notes stay short and outcome-first; **hotfix** gets a dedicated place for *what broke / what we patched / why now* without turning every tier into a postmortem.
- **Evolvability:** The tier is **optional** (no file → no menu entry for that version), so normal releases stay uncluttered.
- **Shared cue:** The larger system and **MEMORY** anchor point to the skill and editorial guide so “how we write hotfixes” does not live only in chat.
