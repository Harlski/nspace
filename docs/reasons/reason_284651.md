# Reason 284651 — Build dock color RGB, hex popover, live preview

**Updated:** [docs/THE-LARGER-SYSTEM.md](../THE-LARGER-SYSTEM.md) (*Authoring HUD: bottom build dock*).

## Why this change

Nimiq Space moved block and gate tinting from a fixed **palette index** to **`colorRgb`** so authors can pick any hue and type **#RRGGBB**. That touches **wire format**, **server load migration**, and **three HUD surfaces** (placement, selection edit, room sky). Without a durable note, future work might reintroduce **recent swatches**, split color UX per surface, or wire **live hex preview** through **`syncBuildHud`** again—which previously closed the popover and risked feedback loops with **`setBuildBlockBarState`**.

## How it serves the larger system

- **Coherence:** One mental model—**hue ring + center hex**—for object color and room ambient color, aligned with **Objects vs Room** scope.
- **Evolvability:** **`colorRgb`** is the forward contract; legacy **`colorId`** stays a load-time migration, not a parallel authoring path.
- **Authoring quality:** **Live preview** updates local UI and game meshes only; **commit** paths persist (placement handler, panel `emitPanelProps`, room-bg pick). That keeps the dock responsive and matches selection-edit behavior.
