# reason_920419 — Linked teleporter reposition moves one end only

**Why:** Players reported that moving one teleporter in a same-room linked pair dragged the peer by the same delta. Reposition should affect only the tile being moved.

**How it serves the larger system:** Linked pairs still share `pairedPeerKey` and reciprocal warp targets, but reposition is per-endpoint—matching gate/billboard move semantics and avoiding surprise layout shifts when editing one portal.

**Change:** Replaced atomic pair delta move in `server/src/rooms.ts` with single-endpoint move plus peer target refresh.
