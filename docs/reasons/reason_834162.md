# Reason 834162 — companion rationale for `THE-LARGER-SYSTEM` edits

**Linked change:** [THE-LARGER-SYSTEM.md](../THE-LARGER-SYSTEM.md) — new principle on **client-only visuals** attached to **authoritative** world geometry (placed blocks, future props).

## Why the system was updated

We added **mineable (active claimable) block** sparkle particles in the client. A first pass used `Box3.setFromObject` on the whole obstacle group, which **inflated the build-mode selection wireframe** because particle positions sat on a larger shell than the block mesh. Separately, depth-tested particles were often **invisible** from isometric angles because fragments lost to the block’s own depth buffer.

Fixing that established a **repeatable pattern** (skip raycast, skip bounds, solid-mesh-only selection) that will apply to future per-block VFX. That pattern is a **cross-cutting product decision**: cosmetics must not hijack interaction chrome or imply false collision geometry.

## How this better supports THE-LARGER-SYSTEM

- **Coherence:** server authority and WS contracts stay the source of truth; client flourishes are clearly scoped.
- **Evolvability:** new effects can attach under the same `userData` convention without rediscovering depth/picking pitfalls each time.
- **Clear direction:** contributors know selection and picking must key off **solid meshes**, not decorative children, unless deliberately unified.
