# reason_829461 — Mini Apps billboard campaigns and deeplinks

**Date:** 2026-06-12

## Why `THE-LARGER-SYSTEM.md` was updated

We shipped paid Hub billboard campaigns for the Nimiq Mini Apps competition: `/advertise` submission, `nspace.billboard.slot` payment intents, and in-world visit links that open competitor mini-apps via Nimiq Pay deeplinks.

## How this serves the larger system

- Keeps **one contract** for visit targets (HTTPS on the server, deeplink construction client-side in Pay) so billboards stay portable across hosts.
- Separates **monetization** (payment-intent sidecar) from **world authority** (game server placement + expiry), matching existing outbound/inbound NIM boundaries.
- Documents the discovery loop: stand on billboard → Visit pill → competitor mini-app — without ad-hoc per-advert URL schemes in synced state.

## Follow-up (2026-06-12): HTTPS in-WebView, not `nimiqpay://` from inside Pay

Assigning `nimiqpay://miniapp` from within the Nimiq Pay WebView disconnected Nimiq Space without loading the target. Official docs position that scheme for **external** entry (tap link → open Pay). In-world visits now use **HTTPS navigation** so Pay loads the competitor URL in the same shell.
