# Reasons — 0.4.0 (patch-notes version)

**Patch-notes version:** `0.4.0` (frozen via `npm run prepare-merge`).

**Role:** Technical **reasons** attached to **this** patch-notes version — file-level and behavior-level inventory. Public-facing copy at other depths lives only under [public/](public/).

---

## Summary

- **World Cup soccer (seasonal, feature-flagged)** — complete. Server-simulated kickable ball, standalone field room with two goals, proximity (run-into) kicks, country-tagged goal tally with per-player counts and immutable per-goal attribution, in-room scoreboard, leaderboard API, and a build-dock "Ball" prop to place kickaround balls in buildable rooms. Gated behind `WORLDCUP_ENABLED` / `VITE_WORLDCUP_ENABLED` (both default on); isolated under `server/src/worldcup/`, `client/src/worldcup/`, `worldcup/issues/`. First server-simulated dynamic non-player object.
- **Pitch makeover + free movement** — soccer-ball texture, grass/line-markings pitch surface, goal netting, and stadium stands (client-only, `client/src/worldcup/fieldVisuals.ts`); plus straight-line any-direction movement on the field room (first per-room non-pathfinding movement variant) so the ball can be kicked at any angle.
- **Cheering crowd** — client-only animated spectators (`WorldcupCrowd`) on the stadium stands: idle bob, ambient synchronized wave cheers, emoji bubbles, and a full eruption on every goal (`goalScored` → `game.worldcupCrowdCheer()`). Server fake-player wanderers are disabled on the pitch so the field stays clear of NPCs.
- **Daily (UTC) scoreboard + champion crowd** — scoring buckets per UTC day and resets at 00:00 UTC; completed days are archived to `history` (nothing lost) and the previous day's winning country becomes the flag the crowd waves (held-up banners + flag-biased cheer emoji). Country *choices* persist across resets; only goal counts reset. Reset is broadcast in-process on the room tick. Legacy cumulative score files migrate to a `0000-legacy` history entry.
- **Goalies + goal rewards (World Cup)** — every goal in the **Free Play Field** is now defended by a server-controlled **Goalie** that tracks the ball laterally with reaction lag and a capped move-speed (A/B `WORLDCUP_GOALIE_MODE`: `kicker` clears the ball via the existing proximity-kick path, `blocker` is a physical collider). Scoring there queues a small NIM payout (default 0.25 NIM) to the credited scorer through the existing payout queue, wrapped in layered env-tunable anti-farming guards (deterministic idempotent `claimId`, per-wallet daily cap, global daily budget, **Contested** ≥2-player requirement; a goal off a Goalie deflection or past a guard still counts for the leaderboard but pays nothing). Private 1v1 **Matches never pay**. Pure, fully unit-tested core (`goalReward`, `match`, `goalie`, blocker-mode physics). See `worldcup/adr/0002` + `docs/reasons/reason_562094.md`.
- **1v1 Match loop end-to-end + own-goal credit reversal + Goalie identicon** — the Match flow now ships behind the hexagonal Action Wheel. A player raises an open **Challenge** ("Open to 1v1") that floats a bubble above their avatar; a nearby player **accepts** from the other-player context menu, and the server spins up an ephemeral **Match Pitch** (field template, never persisted — see ADR 0001), teleports both in, runs the tested `match` reducer broadcasting `matchState`/`matchEnded`, then after a result linger returns both home and tears the empty room down (disconnect/leave forfeits to the opponent). A client **Match HUD** shows the clock, both scores, and the result with a Leave button. Match goals score by which net is breached (own goals count for the opponent) and **never pay NIM**. The **goal-credit model is reversed**: a goal that deflects off the Goalie — including a fumbled own-goal — now credits the **last real (human) kicker** (tracked separately from Goalie touches) for the leaderboard **and** the Free Play NIM reward (same guards); only a goal with no recent human touch credits nobody. This supersedes PRD Story 26's "credit nobody" rule and the blocker-mode sentinel-zeroing. The **Goalie** is now drawn as a fixed **House Keeper** identicon billboard + keeper ring (one stable seed for every goal) in both the field and every Match Pitch. **Spectating deferred** to a follow-up. See `worldcup/adr/0002`, PRD Story 26, + `docs/reasons/reason_884213.md`.
- **Action Wheel (hexagonal self-menu)** — right-click / long-press your own avatar now opens a **flat-top hexagon** menu centred on you instead of the flat 5-emoji strip. The transparent hexagonal **Hub** frames your avatar; the hexagon has **six fixed Sectors** (one per edge): a bottom **Nav Sector** is **Close** (root) / **Back** (sub-wheel), **Emotes** sits upper-left and **Games** (⚽) upper-right, and unused edges render as dim, non-interactive **reserved** Sectors so the hexagon always reads whole (and future actions slot into an edge without a layout rewrite). Root Sectors are **icon-only**. **Emotes** is a paged sub-wheel (Back + 4 emotes + a wrap-around **More** Sector) that sends the emote to chat; **Games** wires **Join Free Play Field** (existing World Cup `field` join) plus an **Open to 1v1 / Cancel 1v1** toggle that raises/clears the player's open Challenge (the Match flow it gates now ships — see the 1v1 Match loop entry). Persistent (no auto-close); dismiss via Close / outside-click / Esc / walking; follows the avatar each frame, anchored slightly lower on the body so the avatar sits centred in the Hub. Drawn as inline SVG polygon Sectors (each Sector's straight-edged shape is its own hit target — no conic-gradient/clip-path drift). Pure geometry + paging extracted to a unit-tested module. See `CONTEXT.md` (Action Wheel / Sector / Emote Wheel / Games Wheel / Hub / Nav Sector) + `docs/adr/0001-action-wheel-replaces-emoji-strip.md`.
- **World Cup — Match Polish & Controls** — eight refinements to the seasonal soccer feature, all under the existing flag. (1) **Kickoff Countdown**: accepting a Challenge no longer teleports instantly — both players are locked, show a 🤝 handshake bubble, and see a "Match starting in 3…2…1" overlay in the origin room (`WORLDCUP_MATCH_COUNTDOWN_MS`, default 3s) before the server teleports them onto the Match Pitch; a disconnect/leave during the countdown aborts it. (2) **Spectating** (implements issue 80): a "{identicon} vs {identicon}" **Spectate Portal** spawns in the room a Match started from; clicking it drops an onlooker into the stands as a **Spectator** (fixed seat, excluded from ball physics, can't move) up to `WORLDCUP_MATCH_SPECTATOR_CAP` (portal shows "full" past the cap); a spectator flag distinct from `matchId` means a spectator disconnect never forfeits; spectators auto-return home and the portal is removed when the Match ends. (3) **Flat scoreboard bar**: the Match HUD is rewritten as a horizontal `{flag}{identicon} scoreA · clock · scoreB {identicon}{flag}` bar pinned under the top HUD with a Leave button at the right edge and inline result. (4) **Crowd Allegiance**: the stands split by side on a Match Pitch (west waves side a's flag, east side b's) and reflect the live roster (distinct present countries) on the Free Play Field, falling back to the champion flag. (5) **Touch Joystick**: a *floating* left-thumb virtual stick on touch / Nimiq Pay steers the player continuously on the pitch — it materializes wherever the thumb presses down and drags (no fixed home), emitting throttled `moveTo` toward a far clamped point; coexists with tap-to-move (quick stationary tap walks, drag past ~14px engages the stick) and the steering finger keeps priority over two-finger camera. (6) **Click-toward-tap**: an off-tile click in grid rooms now walks to the nearest walkable tile. (7) **Outfield Margin**: players may step ~1 tile past the pitch edges (`WORLDCUP_FIELD_OUTFIELD_MARGIN`, default 1.0) to get behind a wall-pinned ball; ball collision walls are unchanged. (8) **Player country on the wire**: `matchState`/`matchEnded` now carry `aCountry`/`bCountry` and field-like player snapshots carry `worldcupCountry`, the data the scoreboard bar and crowd split consume.
- **World Cup — scoring difficulty, spectate walk-on & HUD bounds** — three follow-up refinements under the existing flag. (1) **Beatable keeper**: the Goalie default tuning is softened so well-placed corner shots score instead of only flukey deflections — it patrols only the inner part of the mouth (`WORLDCUP_GOALIE_COVERAGE_PCT`, default 60, leaving a permanent corner gap), clears the ball within a shorter kicker-mode reach than a player (`WORLDCUP_GOALIE_KICK_REACH_TENTHS`, default 0.3 vs 0.6), and is slower + laggier (`WORLDCUP_GOALIE_MOVE_SPEED` 4→2.5, `WORLDCUP_GOALIE_REACTION_MS` 180→260); reward guardrails unchanged. (2) **Spectate by walk-on + intent**: clicking the "vs" portal no longer teleports — you walk onto its footprint tile and press a **"Watch"** pill (the door-style proximity/intent path), and a **loading screen** covers the drop into the stands; players entering after the kickoff countdown also get an "Entering the match…" loading screen. (3) **HUD bounded to the game area**: the World Cup scoreboard and the Match HUD bar are mounted inside the `.letterbox` (no longer spilling into the black bars), and the scoreboard is now **tap-to-collapse/expand** with the choice persisted.
- **End-of-day NIM payout flush + pending NIM in stats** — at the end of each UTC day the daily-stats scheduler now (1) sends the Telegram report with the **NIM still pending in the queue** folded into a combined **"Total NIM (sent + pending)"** so all NIM earned that day is counted in that day's stats, then (2) auto-flushes the pending payout queue (each recipient's pending jobs combined into one on-chain transfer, same path as admin "Payout in full"; failed sends stay pending for the worker). Pending NIM was previously invisible in the report (only confirmed `nim_payout_sent` events were counted). The pending snapshot is taken **before** the flush and the report is sent **before** the payout, so the day's earnings are not split across midnight. Gated by `NIM_PAYOUT_DAILY_FLUSH_ENABLED` (default on when the payout sender is configured); the flush runs even when the Telegram report is disabled.
