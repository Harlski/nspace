# Public patch notes — developers (`0.4.3`)

**Audience:** contributors and integrators — APIs, WS messages, module boundaries, notable refactors.  
**Depth:** summarized technical changelog; not a full file list (that stays in [../reasons.md](../reasons.md)).

---

- [CHANGE] **Direct Invite → Play Space.** `directInvite` is now a multi-participant model: `DirectInviteRecord` carries a `participants[]` collection (`{ guestId, displayName, wallet, joinedLobby }`) + `capacity`, and phases collapse to `open` / `closed` / `expired` (no more `claimed`/`lobby`/`starting`/`started`). New store ops: `claimInvite` is additive (idempotent reclaim, cap-gated), plus `removeInviteParticipant`, `markHostLeftLobby`, `closeInvite`. New config `MAX_PLAY_SPACE_OCCUPANTS` (env `DIRECT_INVITE_MAX_OCCUPANTS`, default 8).
- [CHANGE] **HTTP.** `GET /api/invite/redeem/:slug` now multi-claims (mints/reuses a per-browser guest id, rejects `full`/`closed`/`expired`); `POST /api/invite/create` is idempotent per host (returns the live space). `nickname`/`upgrade` operate per-participant.
- [CHANGE] **WS.** `directInviteState` now carries `roster[]`, `occupancy`, `capacity` (dropped `guestDisplayName`/`canStart`); `directInviteError.code` is now `expired` | `full` | `closed` | `not_found` | `disabled`; `cancelDirectInvite` now means "leave the space"; `startDirectInviteMatch` is removed (members use the normal `setChallenge`/accept flow).
- [FIX] **Resume override bug.** On WS connect, an explicit invite-lobby `room` param is honored instead of letting `resume=1` → `resolveResumeLogin` fall back to chamber for fresh guests ([server/src/index.ts](../../../server/src/index.ts)).
- [NEW] **Guest confinement.** Guest sessions (`guest:*`) are forced onto their own Play Space at connect, and `joinRoom`/`enterPortal` are blocked for them ([server/src/rooms.ts](../../../server/src/rooms.ts)).
- [CHANGE] **Match return.** `worldcupReturnEntrant` returns Play Space members (players and spectators) to their space and re-registers them, keeping guests confined and refreshing the roster overlay; in-space Challenges are allowed (`setChallenge` no longer blocks invite lobbies).
- [NEW] **Teardown by slug.** A Play Space is closed + its room deleted only once no connected socket carries its `directInviteSlug` (so it survives active Matches); wired on disconnect, explicit leave, and TTL expiry.
- [SEC] **Guest NIM exclusion.** `evaluateGoalReward` returns `{ pay:false, reason:"guest" }` for `guest:*` scorers (`isGuestWallet`); covered by a unit test.
- Client: invite entry connects without `resume`; lobby overlay reworked to a roster view (`occupancy/capacity`, no Start button, share/QR for any occupant); Challenge UI enabled inside Play Spaces ([client/src/invite/](../../../client/src/invite/), [client/src/main.ts](../../../client/src/main.ts)).
- [CHANGE] **Action Wheel tree.** `ActionWheelLevel` expanded to `root|emotes|home|games|soccer|soccer1v1|oneVone` with a breadcrumb stack (`pushActionWheelLevel`/`popActionWheelLevel`) so the shared `soccer1v1` leaf (reached from Games→Soccer→1v1 and the root 1v1 shortcut) backs out to the right parent; a `slotOrderFor`/`placeSlices` helper lays 1–5 actions onto the hex. `showActionWheel` handlers now take `onOpenRooms`, `onOpenPlaySpace` (replaces `onCreateDirectInvite`), `isGuest` (trimmed wheel), and `gamesAvailable` ([client/src/ui/hud.ts](../../../client/src/ui/hud.ts), [client/src/main.ts](../../../client/src/main.ts)).
- [CHANGE] **Share Panel + persistent button.** The lobby overlay is now a dismissible share panel (`isOpen()`, `onClose`, room-code display derived from the slug); HUD gains `setPlaySpaceShareVisible` / `onPlaySpaceShareOpen` for a persistent toolbar button. `main.ts` caches the latest `directInviteState` so the button can re-open the panel; it auto-opens once per space (tracked by slug) and only refreshes on roster updates while open ([client/src/invite/lobbyOverlay.ts](../../../client/src/invite/lobbyOverlay.ts), [client/src/ui/hud.ts](../../../client/src/ui/hud.ts), [client/src/main.ts](../../../client/src/main.ts)).
