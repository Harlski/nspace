# Public patch notes — developers (`0.3.23`)

**Audience:** contributors and integrators — APIs, WS messages, module boundaries, notable refactors.  
**Depth:** summarized technical changelog; not a full file list (that stays in [../reasons.md](../reasons.md)).

---

- **[NEW]** **Usernames** — [`server/src/playerProfileStore.ts`](../../../../server/src/playerProfileStore.ts) (`customUsername`, `usernameChangedAt`, `usernamePromptDeferCount`; **`USERNAME_PROMPT_MAX_DEFERRALS`** = 5, **`USERNAME_COOLDOWN_MS`** = 24h). Policy: [`server/src/usernamePolicy.ts`](../../../../server/src/usernamePolicy.ts) — **`bad-words`** profanity filter + reserved-name set; errors `username_profanity`, `username_restricted`. **`GET /api/player-profile/username-prompt`**, **`POST /api/player-profile/username/defer`**, **`PUT /api/player-profile/username`** (first set allowed when `!hasCustom` even if self-service off; **`skipCooldown`** when no prior custom name). **`POST /api/auth/verify`** may include **`usernamePrompt`**. Admin clear ([`adminClearPlayerUsername`](../../../../server/src/playerProfileStore.ts)) nulls custom name, clears cooldown metadata, resets defer count; banned wallets get `needsPrompt: false`. Client: login gate [`client/src/auth/usernamePromptGate.ts`](../../../../client/src/auth/usernamePromptGate.ts) always fetches fresh prompt status (cached **Enter** included; network failure → retry modal, no silent skip). Modal [`client/src/ui/usernamePromptModal.ts`](../../../../client/src/ui/usernamePromptModal.ts) — **0.5s** Nimiq hex loader only, then identicon + form; **`USERNAME_PROMPT_LOAD_MS`**; defer count under input; **`closed`** flag so defer/save dismisses while busy. Styles [`.username-prompt-*`](../../../../client/src/style.css) — explicit **`[hidden]`** so flex panels do not show loader + form together. Top bar via **`setBrandLinksPlayerDisplayName`** ([`client/src/ui/hud.ts`](../../../../client/src/ui/hud.ts)); **`npm` dependency `bad-words`** on server workspace. **Route order:** register **`username-prompt`** / **`username/defer`** before **`GET /api/player-profile/:address`** so Express does not treat `username-prompt` as a wallet id.
- **[FIX]** `loadRoomRegistry` (`server/src/roomRegistry.ts`): read `isPublic` when `version >= 3` and `deletedAt` when `version >= 4`. `persistRoomsFile` writes v6; v6 entries were previously treated like legacy v1/v2 (public by default, never deleted).
- **[CHANGE]** `createRoom` WebSocket message: `displayName` is required (non-empty after trim). Server no longer falls back to `defaultRoomDisplayName`; clients should always send an explicit name.
- **[CHANGE]** Create-room UI (`client/src/main.ts`): empty name field on open; client-side validation before send.
- **[NEW]** WebSocket query **`resume=1`**: server `resolveResumeLogin` (`server/src/rooms.ts`) reads `playerLastSessionStore` (`PLAYER_RECONNECT_GRACE_MS` = 10 min). Validates `hasRoom` + `isWalkableForRoom`; fallback `CHAMBER_DEFAULT_SPAWN`. Client: `ConnectGameWsOptions.resume` in `client/src/net/ws.ts`; login/reconnect paths in `main.ts` use resume; door/teleport/Return Home still send explicit `room` + `sx`/`sz`.
- **[NEW]** Disconnect handler saves `{ roomId, x, z, y?, disconnectedAt }` per wallet (skipped for stream observers). Store file default `server/data/player-last-sessions.json`; env **`PLAYER_LAST_SESSION_STORE_FILE`**.
- **[NEW]** Chat line context menu (`client/src/ui/hud.ts` `openChatLineContextMenu`): top-level **View Profile**, **Report Message** (read-only quoted message + reason → **`POST /api/feedback`** with `source: "report"` and structured **`report`** object), **More** submenu. **`WorldContextMenuItem.children`** + submenu panel in `client/src/ui/worldContextMenu.ts` (hover/focus; click toggle on touch).
- **[NEW]** Feedback tickets — store [`server/src/feedbackTicketStore.ts`](../../../../server/src/feedbackTicketStore.ts) (`FEEDBACK_STORE_FILE`, default `server/data/feedback/tickets.json`; `lastReadAtMs` per ticket; `wallet` on player/admin summaries). Player: **`POST /api/feedback`** (`kind`, `message`, optional `source`), **`GET /api/feedback/mine`** (`unreadCount`, per-ticket `unread`), **`GET /api/feedback/:id`** (marks read), **`POST /api/feedback/:id/messages`**. Admin: **`/admin/feedback`** ([`server/src/adminFeedbackPage.ts`](../../../../server/src/adminFeedbackPage.ts)) — inbox → detail (newest messages first); submitter identicons via **`GET /api/identicon/:wallet`**; **`GET/PATCH /api/admin/feedback`**, **`POST …/messages`**, **`POST …/reward`** (`amountNim` 0.1–2 when status **integrated** → `enqueueNimPayout` `claimId` `feedback-{id}`). Client: self-profile **Feedback** button + top-bar **`.hud-player-bar--feedback-unread`** dot (`refreshFeedbackUnreadBadge`, 45s poll); overlay header grid — **Send**/**Reply** left, centered **New** / **My feedback** tabs, **Close** right; `setFeedbackHandlers` in [`client/src/ui/hud.ts`](../../../../client/src/ui/hud.ts); wired in [`client/src/main.ts`](../../../../client/src/main.ts).
