# 01 — Oversized in-game HUD buttons on production

## Report

On the **live server**, in-game controls such as **Reconnect**, the **teleporter / portal enter** pill, and similar Nimiq-styled pills render **much larger** than intended. **Localhost dev is fine.** Screenshot captured while disconnected (`ws: closed`) with an oversized Reconnect pill in the top strip.

## Triage notes

> *This was generated by AI during triage.*

**Redundancy check:** No existing fix for in-world HUD pill sizing on production-only deploys. Related prior work: `client/src/nimiqRootRemReset.css` documents production CSS load-order issues with `@nimiq/style` setting `html { font-size: 8px }`, which shrinks rem-based UI — but this bug presents as **oversized** pills, suggesting incomplete overrides on specific HUD controls rather than the login-menu shrink case.

**Prior rejection:** None in `.out-of-scope/`.

**Codebase summary:**

- Reconnect: `hud-reconnect-btn nq-button-pill light-blue` in the HUD top strip (`setReconnectOffer`).
- Portal enter: `hud-portal-enter nq-button-pill` — has explicit `min-height: 0; height: auto` override.
- Reconnect has px `font-size: 11px` but **lacks** the same pill compaction overrides as portal-enter and `.hud-top-toolbar .hud-mode-feedback.nq-button-pill`.
- `@nimiq/style` `.nq-button-pill` uses rem-based sizing and large `::before`/`::after` touch-target pseudos; `style.css` disables pointer-events on pseudos under `#app` but visual size may still inherit Nimiq defaults where not overridden.
- `nimiqRootRemReset.css` is imported immediately after Nimiq style in `nimiqStyleShell.ts` to keep `html` at 18px in production bundles — verify it is present and ordered correctly in the **built** live assets.

**Verification:** Not reproduced in this triage session (requires live deploy). Hypothesis: production bundle CSS order or missing scoped overrides on reconnect + other unscoped in-game `nq-button-pill` instances.

**Recommendation:** `bug` + `ready-for-agent`.

## Agent Brief

**Category:** bug  
**Summary:** In-game Nimiq pill buttons (Reconnect, portal enter, similar HUD CTAs) must match compact localhost sizing on production builds.

**Current behavior:** On the live deployment, several in-game `nq-button-pill` controls appear visually oversized (large pill height/padding relative to the top chrome). Local dev (`npm run dev`) renders them at the intended compact size.

**Desired behavior:** All in-game HUD pills use consistent compact sizing on production and localhost. Touch targets remain usable but must not dominate the top strip. Fix should survive `@nimiq/style` load order in split CSS production bundles.

**Key interfaces:**

- HUD reconnect control — compact pill styling in the top strip
- Portal enter pill — reference implementation that already constrains `nq-button-pill` size
- Nimiq style shell import chain — ensures root rem reset loads after `@nimiq/style` in production
- Any other in-game `nq-button-pill` usages missing compaction rules (audit under `#app`)

**Acceptance criteria:**

- [ ] Reconnect button on live matches localhost compact size (visual check on deployed build)
- [ ] Portal enter and other in-game HUD pills remain compact on live
- [ ] No regression to main-menu / login Nimiq buttons
- [ ] `npm run build` passes
- [ ] Document in issue comments which CSS rule(s) fixed prod if load-order related

**Out of scope:**

- Redesigning the top chrome layout
- Non-pill HUD elements unrelated to `@nimiq/style`

## Comments

### 2026-06-21 — implemented

**Root cause:** Vite production splits CSS into two `<link>` tags. `style.css` loads first; `@nimiq/style` + `nimiqRootRemReset.css` load second in `main-*.css`. Single-class HUD overrides (`.hud-reconnect-btn`) tied with `.nq-button-pill` on specificity and lost for `height`, `line-height`, and `font-size` — Nimiq defaults are `3.375rem` / `1.75rem` (~61px tall at 18px root). Dev bundles CSS in import order so `style.css` already won.

**Fix:** Added `#app .… .nq-button-pill` compact sizing block in `client/src/style.css` (specificity beats `.nq-button-pill` regardless of chunk order). Covers reconnect, portal enter, and mode-feedback pills.

**Verify on live** after deploy: Reconnect pill height should match localhost top chrome.
