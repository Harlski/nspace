# 07 — Profile / wardrobe / achievements toggle spam causes lag and server hits

## Report

Severe game lag when spamming **C** to open/close the self profile (wardrobe). Same when opening the achievements menu (**Y**). Suspected too many server hits. Idle FPS drops from ~120 to 60–120 after opening wardrobe once, even when standing still.

## Triage notes

> *This was generated by AI during triage.*

**Verification:** Confirmed via code path review and client unit tests (`api.test.ts`, `panelOpenFetch.test.ts`).

**Root causes:**

1. Each profile reopen refetches `/api/player-profile/…` and `/api/cosmetics/wardrobe` even when cached.
2. Achievement panel open has no in-flight fetch coalescing or session cache.
3. Closing self-profile hides wardrobe but leaves the isolated WebGL preview RAF loop running.

**Recommendation:** `bug` + `ready-for-agent`.

---

## Agent Brief

**Category:** bug  
**Summary:** Stop profile/wardrobe/achievements toggle spam from amplifying server hits and leaking a background WebGL render loop.

**Current behavior:**

- Toggling self-profile (**C**) refetches player profile and wardrobe on every open.
- Toggling achievements (**Y**) fires a new achievements fetch every open with no in-flight coalescing.
- Closing self-profile hides the wardrobe panel but leaves the wardrobe avatar preview WebGL context running.

**Desired behavior:**

- Reopening self-profile within a session should not hit the server unless data is stale or explicitly invalidated.
- Achievement fetches should coalesce in-flight requests and reuse a short-lived session cache.
- Closing self-profile must dispose/pause the wardrobe preview renderer so idle FPS returns to baseline.

**Acceptance criteria:**

- [ ] Spamming wardrobe fetches within cache window does not multiply HTTP requests
- [ ] Achievement opens coalesce in-flight fetches; rapid re-open uses cache
- [ ] Closing self-profile disposes wardrobe preview WebGL (no background RAF)
- [ ] Profile network fetch skipped when session cache is fresh
- [ ] Loadout/unlock mutations invalidate wardrobe cache; unlock refresh forces achievements fetch
- [ ] Regression tests updated and passing

**Out of scope:**

- Server-side rate limiting
- Broader idle FPS unrelated to wardrobe preview

## Comments

**2026-06-30 — implemented**

- Session cache + in-flight coalesce on `fetchWardrobe` / `fetchMyAchievements` (60s TTL, `force` + `invalidate*`).
- Self-profile close calls `disposePreviewCanvas()`; profile fetch skipped when cache fresh; wardrobe reopen skips refresh when data loaded.
- Achievement unlock invalidates achievements cache; cosmetic reward unlock also invalidates wardrobe cache.
- Tests: `cosmetics/api.test.ts`, `achievements/api.test.ts`, `achievements/panelOpenFetch.test.ts`.
